# Session summary — 2026-06-28/29

This session took the astimate idea from a blank repo to a **validated H6**:
the LSP/RAG gap is real, and an astimate-style symbol index fills it.
Two full sweeps (bunker-flash / Qwen 27B and GLM-5.2) both reached
`VERDICT: H6 SUPPORTED`, with GLM-5.2 removing the Q-B (model) noise that
clouded the first sweep.

## Headline result (GLM-5.2, 150 measured runs)

H6 decision view — **Type C+D (the LSP/RAG gap)**, 25 samples each:

| Cond | trimmed_tok | median_tok | acc | done |
|------|------------:|-----------:|----:|----:|
| C0 baseline | 11,014 | 7,451 | 0.60 | 1.00 |
| C1 RAG | 20,453 | 18,006 | 0.60 | 1.00 |
| **C2 astimate** | **7,776** | **6,140** | **0.80** | **1.00** |

- C2/C1 trimmed-token ratio = **0.38** (rule threshold < 0.70 → pass)
- C2 accuracy **higher** than C1 (no accuracy-traded-for-tokens suspicion)
- **VERDICT: H6 SUPPORTED**

Per-type, C2 wins or ties accuracy on **every** type (A 1.00, B 1.00, C 1.00,
D 0.50=tie, E 1.00) at the lowest or near-lowest token cost. The Type E
(loc-slicing) regression seen under bunker-flash (acc 0.40) fully recovered
to 1.00 — confirming that regression was a Q-B (model) artefact, not a
format (Q-A) flaw.

## Honest limitations (from the GLM analysis)

1. **Single codebase (ripgrep).** Generalization to other repos / languages
   is unverified — the strongest remaining threat.
2. **Type D accuracy is a tie, not a win.** astimate's advantage on
   dependency-context tasks is *efficiency* (≈half the tokens), not *accuracy*.
3. **C0 baseline is weak on Type A (acc 0.10)** — tasks are hard, which
   inflates C2's relative lead somewhat (though Type E shows the lead is real).
4. **Meaning layer was hand-authored for the 12 answer symbols.** Generation
   quality at scale (Q-B at the *index-build* step) is not yet exercised.
